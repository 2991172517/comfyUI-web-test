# 打包与闭源分发

本文回答：**不想把源码给对方时，还能怎么发 CustomProject？**  
（仍只发控制台插件，**不含 ComfyUI**。）

---

## 一、先认清：什么算「源码」

| 部分 | 开发态 | 构建后对方能看到什么 |
|------|--------|----------------------|
| **前端** `frontEnd/src/` | Vue 源码 | `npm run build` → `dist/` 里是打包压缩后的 JS/CSS，**一般不必给 src** |
| **后端** `backEnd/*.py` | Python 源码 | 直接 zip = **明文 .py**；需额外步骤才能「运行包」 |
| **数据** `config/`、`workflows/`、`prompt/` | JSON | 无论是否闭源，通常都要给对方（可编辑的配置与工作流） |

所以：**只 build 前端、仍把整个 `backEnd/` 打 zip，等于后端源码照样送出。**  
闭源分发 = **前端只发 dist + 后端打成可执行文件或容器**，不要发 `.py` / `frontEnd/src/`。

---

## 二、分发形态对比

| 形态 | 包内有什么 | 是否算给源码 | 对方环境 | 启动方式 |
|------|------------|--------------|----------|----------|
| **D0 全源码** | `backEnd/` + `frontEnd/src/` | 是 | Python + Node | ComfyUI + `python main.py` + `npm run dev` |
| **D1 半构建（常见误区）** | `backEnd/` + `frontEnd/dist/` | **后端仍是源码** | Python +（可选 Node preview） | 三进程 |
| **D2 闭源运行包（推荐闭源）** | `api.exe` 或 Docker 镜像 + `frontEnd/dist/` + 数据目录 | **否**（仅字节码/二进制） | ComfyUI；Python 可不要* | ComfyUI + 双击 exe / `docker run` + 静态页 |
| **D3 你托管 API** | 只发 `frontEnd/dist/` 或静态页 | 后端在你服务器 | 对方只有 ComfyUI | 对方浏览器连你的 HTTPS API |

\* PyInstaller 单文件 exe 内嵌 Python 运行时，对方不必单独装 Python；ComfyUI 仍需要对方自己的 Python。

### 怎么选

- **同事内测、信任环境** → D1 即可（省事，后端仍是 .py）。
- **不想给后端逻辑、给固定用户用** → **D2**。
- **强控制、可联网** → D3（运维成本高，ComfyUI 仍在对方机器）。

---

## 三、D2 闭源运行包：推荐目录结构

解压后放进 ComfyUI 根目录：

```text
ComfyUI/
  CustomProject/
    bin/
      customproject-api.exe      # Windows；或 Linux 无后缀可执行文件
    frontEnd/
      dist/                      # 仅构建产物，无 src
    config/
    workflows/
    prompt/
    docs/
      使用说明.md
    start.bat                    # 可选：一键起后端 + 打开浏览器
```

**不要包含：**

- `backEnd/*.py`、`backEnd/routers/`、`backEnd/vocabulary/` 等源码树  
- `frontEnd/src/`、`frontEnd/node_modules/`  
- `scripts/`（维护脚本，非运行必需）

**仍建议包含（或让对方自行配置）：**

- `config/` — 登录、预设、全局提示词（可脱敏后发放）  
- `workflows/` — API 工作流  
- `prompt/jsonData/manifest.json` — tag 词库（体积大时可单独包）

## 四、后端打包：PyInstaller（已内置脚本）

### 4.1 一键构建（推荐）

在 **CustomProject 根目录** PowerShell 执行：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/build_release.ps1
```

会依次：

1. `npm run build` → `frontEnd/dist/`
2. `pyinstaller customproject-api.spec` → `backEnd/dist/customproject-api/`
3. 复制到 `bin/`，并组装 `release/CustomProject/`（**无 .py、无 src**）
4. 生成 `release/CustomProject-closed.zip`

对方解压到 `ComfyUI/CustomProject/`，先开 ComfyUI，再双击 **`start-api.bat`**，浏览器打开 **http://127.0.0.1:8000**（后端已托管前端 dist，不必再开 Vite）。

仅重打前端、跳过 PyInstaller：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/build_release.ps1 -SkipPyInstaller
```

### 4.2 手动 PyInstaller

```powershell
cd ComfyUI\CustomProject\backEnd
pip install -r requirements.txt pyinstaller
pyinstaller --noconfirm --clean customproject-api.spec
```

产物：`backEnd/dist/customproject-api/customproject-api.exe`（目录版，启动较快）。  
将整个 `customproject-api` 文件夹内容拷到 `CustomProject/bin/`。

### 4.3 运行包目录结构

```text
CustomProject/
  bin/
    customproject-api.exe      # 及 PyInstaller 依赖 dll、pyd
  frontEnd/dist/               # 仅构建产物
  config/ workflows/ prompt/
  data/                        # 首次运行自动创建（词库索引等）
  start-api.bat
```

**不要**包含 `backEnd/*.py`、`frontEnd/src/`。

### 4.4 环境变量（可选）

| 变量 | 默认 | 说明 |
|------|------|------|
| `COMFYUI_ROOT` | `CustomProject` 的上级目录 | ComfyUI 安装根 |
| `CUSTOM_PROJECT_ROOT` | `CustomProject` 目录 | 含 config、workflows |
| `COMFYUI_URL` | `http://127.0.0.1:8188` | ComfyUI API |
| `API_PORT` | `8000` | 控制台端口 |
| `SERVE_FRONTEND` | `auto` | `auto`=有 dist 则托管；`0`=仅 API |

### 4.5 PyInstaller 说明

**优点**：对方无需 `pip install`、看不到 `.py` 源码。  
**缺点**：体积约 **30–50 MB**（本仓库实测 `bin/` 约 36 MB）；**不能防专业逆向**；换 Python 版本需在你机器上重打。

配置文件 `backEnd/customproject-api.spec` 已收集 `routers`、`vocabulary`、`model_parser` 等子模块；若打包后某 API 报 `ModuleNotFoundError`，在 spec 的 `hiddenimports` 中补充后重打。

## 五、其它后端打包方式（简述）

### 5.1 Nuitka

```powershell
pip install nuitka
python -m nuitka --standalone --follow-imports backEnd/main.py
```

配置比 PyInstaller 复杂，适合对体积/逆向有更高要求时投入。

### 5.2 Docker（跨平台、易版本锁定）

镜像内只跑 **CustomProject API**，通过卷挂载 ComfyUI：

```yaml
# 示例 docker-compose.yml（放在 CustomProject 根目录）
services:
  customproject-api:
    image: your-registry/customproject-api:1.0
    ports:
      - "8000:8000"
    environment:
      COMFYUI_URL: "http://host.docker.internal:8188"
    volumes:
      - ../models:/comfy/models:ro          # 按实际 ComfyUI 路径调整
      - ../output:/comfy/output
      - ./config:/app/config
      - ./workflows:/app/workflows
      - ./prompt:/app/prompt
```

对方需装 Docker Desktop；ComfyUI 仍在宿主机跑。适合 Linux 服务器或统一运维环境。

### 5.3 后端静态托管前端（已内置）

exe / `python main.py` 在检测到 `frontEnd/dist/index.html` 时，会在 **8000 端口** 同时提供 API 与网页（`SERVE_FRONTEND=auto`）。  
闭源包只需 **ComfyUI + start-api.bat** 两个进程。

## 六、前端：闭源必做

```powershell
cd frontEnd
npm ci
npm run build
```

分发 **`frontEnd/dist/` 整目录**，删除或不要打包：

- `frontEnd/src/`
- `frontEnd/node_modules/`

`dist/assets/*.js` 为压缩混淆后的 bundle，足够应对一般「不要源码」需求。

---

## 七、路径与环境变量（exe 必看）

源码版 `backEnd/config.py` 在 **exe 模式** 下自动推断：

- `CUSTOM_PROJECT_ROOT` = exe 的上级的上级（`CustomProject/`）
- `COMFYUI_ROOT` = 再上一级（`ComfyUI/`）

也可手动设置环境变量覆盖（见 4.4 表）。`start-api.bat` 已写入常用变量。

---

## 八、闭源包的打包清单（D2）

**包含：**

- [ ] `bin/customproject-api.exe`（或 Docker 镜像名写进说明）  
- [ ] `frontEnd/dist/`  
- [ ] `config/`（脱敏后的 auth、预设）  
- [ ] `workflows/`、`prompt/`（按需）  
- [ ] `docs/使用说明.md`（改启动命令为 exe）  
- [ ] 可选 `start.bat` / `start.ps1`

**排除：**

- [ ] 整个 `backEnd/` 源码树（除非法律/许可要求保留 LICENSE）  
- [ ] `frontEnd/src`、`node_modules`  
- [ ] `.git`、`__pycache__`、`.venv`  
- [ ] `civitai_token.txt`、真实密码

## 九、现实预期（诚实说明）

1. **Python 没有 Java 那种强封闭**；PyInstaller/Nuitka 只能提高门槛，无法防 determined 逆向。  
2. **业务逻辑在服务端**：若极度敏感，考虑 **D3 自建 API**，ComfyUI 仅作本地算力（架构改动大）。  
3. **JSON 配置与工作流**对方总能看到内容，这与「代码闭源」无关，属于数据层。  
4. **ComfyUI 与模型**永远不在你包里，对方本机仍可改模型、抓 ComfyUI 请求。
