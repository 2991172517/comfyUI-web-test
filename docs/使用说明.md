# CustomProject 使用说明（分发给他人）

本文面向**使用者**（非开发者）：如何安装、启动、登录与日常生图。

## 分发范围（重要）

**只打包、只分发 `CustomProject` 这一层功能**，不包含 ComfyUI 本体。

| 你发给对方的 zip 里有什么 | 不在包内、对方需自备 |
|---------------------------|---------------------|
| `CustomProject/` 整目录（后端、前端、config、workflows、词库等） | **ComfyUI**（官方/整合包/Aki 等，自行安装） |
| 可选：预构建的 `frontEnd/dist/` | **Checkpoint / LoRA 模型**（放在 ComfyUI 的 `models/`） |
| 控制台用的工作流 JSON、提示词预设、tag manifest | 显卡驱动、Python 环境（通常随 ComfyUI 已有） |

对方的使用方式是：**先有自己的 ComfyUI → 把 zip 解压出的 `CustomProject` 文件夹放进 ComfyUI 根目录 → 再按本文启动控制台**。

控制台不能脱离 ComfyUI 单独生图；它只是一个「编排 + 网页 UI」插件层。

---

## 一、推荐的分发方式

根据对方技术水平，选一种即可。**三种方式都只发 CustomProject，不发 ComfyUI。**

| 方式 | 适合谁 | 对方除 ComfyUI 外还要装什么 | 启动几个进程 |
|------|--------|----------------------------|--------------|
| **A. 源码 + 开发模式** | 会改前端/后端的人 | Python 依赖 + Node.js | 3（ComfyUI + 后端 + `npm run dev`） |
| **B. 半构建包** | 普通用户 | Python 依赖；Node 仅 preview | 3；**后端仍是 .py 源码** |
| **C. 闭源运行包** | 不想给源码的用户 | 一般不需 Python/Node* | ComfyUI + exe（+ 浏览器） |

\* 闭源包用 PyInstaller 等把后端打成 exe，详见 **[打包与闭源分发.md](./打包与闭源分发.md)**。

**若不想把源码给对方，不要只发「B」**——B 只是前端 build 了，**`backEnd/*.py` 仍是明文**。应使用 **C：前端 dist + 后端 exe/Docker**，见专文。

### B. 半构建包（仍含后端源码，省事但不闭源）

**最常用、但等于给了后端源码。**

1. 你在打包前于 **CustomProject 目录内** 执行 `npm run build`（产物在 `frontEnd/dist/`）。
2. 将 **整个 `CustomProject` 文件夹** 打成 zip（见下方命令），**不要**包含 `node_modules`、`.venv`、`__pycache__`。
3. 对方解压后，把 **`CustomProject` 文件夹** 复制/合并到**已有 ComfyUI 安装目录**下（与 ComfyUI 的 `main.py` 同级）。
4. 日常启动：先开 ComfyUI → 再开 CustomProject 后端 → 再开前端（见「日常启动」）。

> **不要**把 `config/civitai_token.txt`、个人 `auth_store.json` 里的真实密码等打进包；见 [文件与数据说明.md](./文件与数据说明.md) 的「分发前检查清单」。

### 打包命令示例（PowerShell，在 CustomProject 根目录执行）

```powershell
# 当前目录应为：...\ComfyUI\CustomProject

# 1. 构建前端（方式 B 必做）
cd frontEnd
npm ci
npm run build
cd ..

# 2. 打 zip：只含 CustomProject，解压后应得到 CustomProject\backEnd\ ... 这一层
$parent = Split-Path (Get-Location) -Parent   # ComfyUI 根目录
$dest = Join-Path $parent "CustomProject-release.zip"
if (Test-Path $dest) { Remove-Item $dest }

# 在 ComfyUI 根目录下压缩「CustomProject」文件夹本身（不要压缩整个 ComfyUI）
Compress-Archive -Path (Join-Path $parent "CustomProject") -DestinationPath $dest -Force
```

若 zip 体积过大，可改用「选择性打包」（仍只发 CustomProject，不含 ComfyUI）：

```powershell
$staging = Join-Path $env:TEMP "CustomProject-pack"
$packRoot = Join-Path $staging "CustomProject"
if (Test-Path $staging) { Remove-Item $staging -Recurse -Force }
New-Item -ItemType Directory -Path $packRoot -Force | Out-Null

$items = @(
  "backEnd", "config", "workflows", "prompt", "docs", "scripts",
  "README.md", "START.md",
  "frontEnd\dist", "frontEnd\package.json", "frontEnd\package-lock.json", "frontEnd\vite.config.js"
)
foreach ($i in $items) {
  Copy-Item -Path $i -Destination $packRoot -Recurse -Force -ErrorAction SilentlyContinue
}
# 方式 A 需改界面时，改为复制整个 frontEnd（含 src），不要复制 node_modules

Compress-Archive -Path $packRoot -DestinationPath $dest -Force
Remove-Item $staging -Recurse -Force
```

若对方需要改界面，应发**完整 `frontEnd` 源码**（方式 A），仍不要包含 `node_modules`。

### C. 闭源运行包（不给源码）

步骤见 **[打包指南.md](./打包指南.md)**。一键命令：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/build_release.ps1
```

产物 `release/CustomProject-closed.zip` 发给客户 → 解压到 `ComfyUI/CustomProject/` → `start-api.bat` → http://127.0.0.1:8000

## 二、安装位置（对方操作）

对方**已有 ComfyUI** 的前提下，把 zip 里的 `CustomProject` 放到 ComfyUI 根目录：

```text
对方电脑上的 ComfyUI/          ← 对方自己安装的，不在你的 zip 里
  main.py
  models/                       ← 对方的模型
  output/                       ← 对方的输出
  CustomProject/                ← 你发的 zip 解压/复制到这里
    backEnd/
    frontEnd/
    workflows/
    config/
    prompt/
    docs/
```

**错误示例**：把 `backEnd`、`config` 直接解压在 ComfyUI 根目录（缺少 `CustomProject` 这一层）→ 后端路径会错乱。  
**错误示例**：zip 里带了整个 ComfyUI → 体积巨大且没必要；只发 CustomProject 即可。

后端通过 `backEnd/config.py` 自动把 **ComfyUI 根目录** 定为 `CustomProject` 的上两级。  
若放错层级，会找不到 `output/`、`models/`，健康检查也会失败。

## 三、环境要求

| 组件 | 建议版本 |
|------|----------|
| 操作系统 | Windows 10/11（文档以 PowerShell 为例；Linux/macOS 命令类似） |
| ComfyUI | 已能本地 `python main.py` 正常出图 |
| Python | 3.10+（与 ComfyUI 同环境或独立 venv 均可） |
| Node.js | 方式 A 必需 18+；方式 B 仅预览时需要（或你代为 build 后可不装） |
| 显卡 / 模型 | 由 ComfyUI 侧决定；控制台不 bundled 模型 |

## 四、首次安装（在 CustomProject 内操作）

以下命令路径均在 **`ComfyUI\CustomProject`** 下，不是 ComfyUI 根目录。

### 0. 确认 ComfyUI 已就绪

对方需**自行安装并验证** ComfyUI 能正常 `python main.py` 出图；这一步不在 CustomProject 包内。  
模型放在 ComfyUI 的 `models/checkpoints/`、`models/loras/`；工作流 JSON 已随 CustomProject 在 `workflows/`（也可让对方自行替换）。

### 1. 安装后端依赖

```powershell
cd ComfyUI\CustomProject\backEnd
python -m pip install -r requirements.txt
```

### 2. 安装前端依赖

**方式 A（开发）：**

```powershell
cd ComfyUI\CustomProject\frontEnd
npm install
```

**方式 B（已含 dist）：** 若只跑 `npm run preview`，仍需：

```powershell
cd ComfyUI\CustomProject\frontEnd
npm ci
```

（`npm ci` 比 `install` 更适合按 lock 文件复现依赖。）

### 3. 配置登录（首次必做）

编辑 `CustomProject/config/auth_store.json`：

- **管理员**：`admins` 数组，默认可能有账号 `admin`（若包内仍是示例，请**立即改密码**或禁用）。
- **邀请码用户**：`invites` 数组；可设 `max_uses`、`single_quota_per_login`（每次登录可生成张数）。

也可登录管理员后，在页面 **邀请码管理** 中维护邀请码。

## 五、日常启动

### 端口一览

| 服务 | 地址 |
|------|------|
| ComfyUI | http://127.0.0.1:8188 |
| 控制台后端 | http://127.0.0.1:8000 |
| 控制台页面 | http://127.0.0.1:5173（dev/preview） |

三个窗口都保持运行，直到结束使用。

### 终端 1 — ComfyUI

```powershell
cd ComfyUI
python main.py
```

### 终端 2 — 后端

```powershell
cd ComfyUI\CustomProject\backEnd
python main.py
```

看到 `Uvicorn running on http://127.0.0.1:8000` 即成功。  
可访问 http://127.0.0.1:8000/docs 查看 API。

### 终端 3 — 前端

**方式 A（开发，改代码即时生效）：**

```powershell
cd ComfyUI\CustomProject\frontEnd
npm run dev
```

浏览器打开终端里显示的地址（一般为 http://127.0.0.1:5173）。

**方式 B（使用已构建的 dist）：**

```powershell
cd ComfyUI\CustomProject\frontEnd
npm run preview
```

同样打开 http://127.0.0.1:5173（端口以终端输出为准）。

### 停止

各终端 `Ctrl+C`。建议顺序：前端 → 后端 → ComfyUI。

## 六、登录与权限

| 角色 | 能力概要 |
|------|----------|
| **管理员** | 全部功能；可删除历史/输出；管理工作流、模型说明、邀请码 |
| **邀请码用户** | 生成（有单张额度限制）；历史/收藏浏览；**不能**删记录、不能进工作流配置/任务计划等 |

登录页选择管理员或邀请码登录。会话失效后重新登录即可。

## 七、基本使用流程

### 1. 生成页（主页）

1. 顶栏确认 **ComfyUI 已连接**（绿/正常状态）。
2. 选择 **工作流**（来自 `workflows/` 下的 API 导出 JSON）。
3. 在 **提示词** 模块编辑正向/负向；可导入 **提示词预设**、配置随机组。
4. 在 **LoRA** 模块选模型与权重；可开启 **LoRA 扫参**（张数由档位数决定）。
5. 页面下方 **生成参数**：Seed、生成张数（1 张即单抽，多张即批量）。
6. 点击底部 **生成**；结果在页内或下方网格中查看。

### 2. 历史记录

- 列表浏览单抽与批量任务；可筛选 Checkpoint / LoRA。
- 管理员：**多选** → 批量删除。
- 点击文字区域进详情；点击图片放大。

### 3. 收藏

在生成结果、历史、批量网格中点 ☆ 收藏；**收藏页**可筛选并用「用此配置生成」跳回生成页。

### 4. 工作流配置（管理员）

- 管理子工作流、Checkpoint / LoRA 链。
- 母版 `First_api.json` 只读，请复制为 `variants/` 下子工作流再改。

### 5. 提示词设置

- **全局提示词**：默认正/负向追加、全局随机组。
- **预设库**：导入/导出整套提示词方案。

### 6. Tag 显示管理

- 基于 `prompt/jsonData/manifest.json` 的标签树。
- 可隐藏分类、喜好排序、用户自定义 tag（写入 `backEnd/data/vocabulary_user.json`）。

### 7. 模型管理 / C 站导入（管理员）

- 本地 Checkpoint / LoRA 说明与预览图（预览图放在 ComfyUI `models/` 下，见文件说明）。
- 可选配置 `config/civitai_token.txt` 使用 Civitai 浏览与下载。

### 8. 任务计划（管理员）

- 将多个批量任务串行执行；配置保存在 `config/batch_tasks.json` 与 `output/campaigns/`。

## 八、常见问题

### 页面显示「ComfyUI 未连接」

1. ComfyUI 是否已启动且无报错。
2. 浏览器能否打开 http://127.0.0.1:8188。
3. 若端口不是 8188，修改 `backEnd/config.py` 中 `COMFYUI_URL` 后重启后端。

### 工作流列表为空

将 ComfyUI **File → Export (API)** 导出的 json 放入：

```text
CustomProject/workflows/
CustomProject/workflows/variants/   ← 子工作流
```

### 前端无法访问 /api

- 开发/预览模式下，Vite 会把 `/api` 代理到 `8000`。
- 确认后端已启动；若改了后端端口，同步修改 `frontEnd/vite.config.js` 里 `proxy.target`。

### 后端端口 8000 被占用

修改 `backEnd/config.py` 的 `API_PORT`，并同步 `frontEnd/vite.config.js` 代理端口。

### 邀请码用户无法批量 / 无法删图

属权限设计；需管理员账号或调整 `auth_store.json` 中邀请码策略。
