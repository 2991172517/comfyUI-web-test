# 闭源 exe 打包（开发者）

只发 **CustomProject**，不含 ComfyUI。客户解压到 `ComfyUI/CustomProject/` 后，先开 ComfyUI，再双击 `start-api.bat`。

客户怎么用 → [使用说明.md](./使用说明.md)

---

## 1. 打包前（必做）

在 `CustomProject` 根目录：

1. 改 `config/auth_store.json`：管理员密码、邀请码（默认 `admin`/`admin` 务必改掉）
2. 删除 `config/civitai_token.txt`（若有）
3. 确认 `workflows/`、`config/`、`prompt/` 是要给客户的版本

---

## 2. 一键打包

在 **`CustomProject` 根目录** PowerShell 执行：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/build_release.ps1
```

脚本会自动：`npm run build` → PyInstaller 打 exe → 组装 zip。

**发给客户的文件：**

```text
release/CustomProject-closed.zip
```

---

## 3. 可选：分步 / 增量

```powershell
# 只重打前端（exe 已有）
powershell -ExecutionPolicy Bypass -File scripts/build_release.ps1 -SkipPyInstaller

# 只重组装 zip（dist 和 bin 都已存在）
powershell -ExecutionPolicy Bypass -File scripts/build_release.ps1 -SkipFrontend -SkipPyInstaller
```

---

## 4. 手动分步（不用脚本时）

```powershell
# 在 CustomProject 根目录

# 前端
cd frontEnd
npm ci
npm run build
cd ..

# 后端 exe
cd backEnd
python -m pip install -r requirements.txt pyinstaller
pyinstaller --noconfirm --clean customproject-api.spec
cd ..

# 复制 exe（整目录，含 _internal）
Remove-Item bin -Recurse -Force -ErrorAction SilentlyContinue
New-Item bin -ItemType Directory | Out-Null
Copy-Item backEnd\dist\customproject-api\* bin\ -Recurse -Force
```

然后手动把 `bin/`、`frontEnd/dist/`、`config/`、`workflows/`、`prompt/`、`docs/` 和 `start-api.bat` 打进 zip（或再跑一遍 `-SkipFrontend -SkipPyInstaller` 只组装）。

---

## 5. 打包后自测

```powershell
cd release\CustomProject
# 先开 ComfyUI，再：
.\start-api.bat
```

浏览器打开 http://127.0.0.1:8000 ，登录并试跑一张图。

---

## 6. zip 里有什么

```text
CustomProject/
  start-api.bat
  bin/customproject-api.exe + _internal/
  frontEnd/dist/
  config/ workflows/ prompt/ docs/
```

**不要包含：** `backEnd/`、`frontEnd/src/`、`node_modules/`、`.venv/`

---

## 7. 打包失败

- 缺 Python 模块：改 `backEnd/customproject-api.spec` 的 `hiddenimports`，重跑第 2 步
- 前端失败：在 `frontEnd` 里单独 `npm ci && npm run build` 看报错
