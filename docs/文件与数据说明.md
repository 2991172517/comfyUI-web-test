# CustomProject 文件与数据说明

本文说明 **CustomProject 包内**各类文件放在哪里、谁读写、分发时要注意什么。  
**ComfyUI 本体及其 `models/`、`output/` 不在本仓库内**，但控制台运行时会读写 ComfyUI 目录下的路径。

路径默认：对方机器上的 `ComfyUI/CustomProject/` + 上级 `ComfyUI/`。

---

## 〇、本 zip 里有什么 / 没有什么

| 在 CustomProject 包内 | 不在包内（对方 ComfyUI 安装里） |
|----------------------|--------------------------------|
| `backEnd/`、`frontEnd/`、`config/`、`workflows/`、`prompt/` | ComfyUI 程序、`main.py`、自定义节点 |
| 控制台工作流、预设、tag manifest | `models/checkpoints/`、`models/loras/` 权重文件 |
| `docs/`、`scripts/` | 运行后生成的 `output/custom_single/`、`custom_batch/` |

分发时 **只压缩 `CustomProject` 文件夹**，不要连同 ComfyUI 整棵目录一起打 zip。

## 一、总览目录树

```text
ComfyUI/
├── models/                          # ComfyUI 模型（非本仓库，但控制台会读）
│   ├── checkpoints/
│   └── loras/
├── output/                          # ComfyUI 输出根目录
│   ├── custom_single/               # 单抽历史与 record.json（运行时生成）
│   └── custom_batch/                # 批量任务目录（运行时生成）
├── user/default/favorites/          # 收藏 JSON（ComfyUI 用户目录下）
│
└── CustomProject/
    ├── backEnd/                     # FastAPI 后端
    │   ├── main.py                  # 入口
    │   ├── config.py                # ComfyUI 地址、端口、路径常量
    │   ├── editable_config.json     # 页面上展示哪些节点字段
    │   ├── requirements.txt
    │   └── data/                    # 运行时/索引数据
    │       ├── vocabulary.db        # 标签 manifest 的 SQLite 索引（可重建）
    │       └── vocabulary_user.json # Tag 管理：隐藏/喜好/用户增删 tag
    │
    ├── frontEnd/                    # Vue 3 前端
    │   ├── src/                     # 源码
    │   ├── dist/                    # npm run build 产物（分发包可只带 dist）
    │   └── vite.config.js           # 开发代理 /api → 8000
    │
    ├── workflows/                   # API 工作流 JSON
    │   ├── First_api.json           # 母版（只读模板）
    │   ├── First_api.meta.json      # 拓扑元数据（LoRA 角色、Style 规则）
    │   └── variants/                # 子工作流（可保存参数）
    │       ├── xxx.json
    │       └── xxx.meta.json
    │
    ├── config/                      # 控制台业务配置（JSON，可手改/备份）
    ├── prompt/                      # 提示词词库源数据
    │   └── jsonData/
    │       └── manifest.json        # 标签分类 + 全量 tag 词条（体积大）
    │
    ├── output/                      # 控制台自有输出（非 ComfyUI output）
    │   └── campaigns/               # 任务计划运行记录
    │
    ├── scripts/                     # 维护脚本（导入 manifest、清理 tag 等）
    └── docs/                        # 文档（本文件、使用说明等）
```

## 二、`config/` 配置文件

| 文件 | 用途 | 读写 |
|------|------|------|
| `auth_store.json` | 管理员账号、邀请码、会话相关持久化 | 后端；**分发勿带真实密码** |
| `auth_codes.json` | 旧版邀请码（可迁移到 auth_store） | 后端，可选 |
| `global_prompt_config.json` | 全局正/负提示词、全局随机组、合并顺序 | 前后端 |
| `prompt_defaults.json` | 旧版全局默认（兼容读取） | 后端 |
| `global_reference_groups.json` | 全局参考 tag 组（抽卡/合并用） | 前后端 |
| `global_reference_groups.seed.json` | 参考组种子示例 | 只读参考 |
| `prompt_presets.json` | 提示词预设库（名称、固定前后缀、随机组） | 前后端 |
| `batch_prompt_config.json` | 批量/生成页「当次提示词」默认 | 前后端 |
| `batch_tasks.json` | 任务计划里保存的批量任务模板 | 前后端 |
| `models_manifest.json` | 模型清单（C 站来源、本地路径、说明目录） | 导入/管理功能 |
| `model_node_defaults.json` | 模型节点默认字段 | 工作流/模型管理 |
| `civitai_favorites.json` | C 站模型收藏（按 API Key 分桶） | C 站浏览 |
| `civitai_token.txt` | Civitai API Token（**勿提交、勿打包**） | 可选，见 `.example` |

修改 JSON 后一般**刷新页面**即可；部分需重启后端。建议改前备份。

## 三、工作流 `workflows/`

| 路径 | 说明 |
|------|------|
| `First_api.json` | 从 ComfyUI **Export (API)** 得到的母版；界面保存参数时会提示另存子工作流 |
| `First_api.meta.json` | 链上 LoRA 角色（character/style）、Style 绕过规则等 |
| `variants/*.json` | 子工作流：LoRA 链、采样、提示词节点等可持久化修改 |
| `variants/*.meta.json` | 对应子工作流的元数据 |

列表中的 **工作流 id** = 文件名（不含 `.json`），例如 `First_api`、`my_variant`。

控制台**不会**把参数写进 ComfyUI 的 `user/default/workflows_api/`，而是写回上述 `CustomProject/workflows/` 路径。

## 四、提示词与「参考词 / Tag 库」

### 4.1 主词库 `prompt/jsonData/manifest.json`

- **内容**：分类树（categories）+ 每条 tag 的中英文、所属分类等。
- **用途**：提示词输入补全、Tag 选择面板、Tag 显示管理。
- **索引**：首次或 manifest 变更后，后端在 `backEnd/data/vocabulary.db` 建 SQLite 索引（可删 db 后自动重建）。
- **体积**：通常很大（数万～数十万 tag）；分发时可整包带上，或用脚本裁剪后再发。

### 4.2 用户侧 Tag 覆盖 `backEnd/data/vocabulary_user.json`

- 用户**隐藏**的分类 id
- **喜好**排序（like/dislike）
- **用户新增/删除**的 tag 记录  
分发时：可带空对象 `{}` 或你的定制偏好；个人偏好不必给其他人。

### 4.3 全局参考组 `config/global_reference_groups.json`

- 与 manifest 独立的一组「参考 tag 池」，用于全局抽卡/合并逻辑。
- 可由 `scripts/seed_global_reference_groups.py` 从种子生成。

### 4.4 预设 vs 全局

| 概念 | 文件 | 说明 |
|------|------|------|
| 全局提示词 | `global_prompt_config.json` | 对所有生成生效的全局层 |
| 预设 | `prompt_presets.json` | 命名方案，生成页可导入 |
| 当次批量默认 | `batch_prompt_config.json` | 打开生成页时的提示词层默认值 |

## 五、ComfyUI 侧路径（不在 CustomProject 仓库内）

| 路径 | 用途 |
|------|------|
| `ComfyUI/models/checkpoints/` | Checkpoint 文件 |
| `ComfyUI/models/loras/` | LoRA 文件 |
| `ComfyUI/output/custom_single/<prompt_id>/` | 单张输出 + `record.json`（历史元数据） |
| `ComfyUI/output/custom_batch/<batch_id>/` | 批量输出、`manifest.json`、`run_config.json`、图片 |
| `ComfyUI/user/default/favorites/favorites.json` | 收藏记录（路径+参数，不复制图片） |

### 模型参考图 / 说明 txt

与模型**同目录**放置（控制台会扫描）：

```text
models/checkpoints/my_ckpt.safetensors
models/checkpoints/my_ckpt/          # 推荐：与 stem 同名的文件夹
  说明.txt 或任意首个 .txt           # 选择框旁 ⓘ 悬停全文
  01.png, cover.jpg                  # 参考图轮播

models/loras/style/foo.safetensors
models/loras/style/foo/
  note.txt
  ref.png
```

也可使用 `模型名.safetensors.png` 单张侧挂图。

`config/models_manifest.json` 记录模型与 C 站链接、本地 `asset_dir` 等，**不替代**实际模型文件。

## 八、运行时生成、可不带在 IS 空目录

以下内容随使用产生，新环境会自动再建：

- `ComfyUI/output/custom_single/`、`custom_batch/`
- `CustomProject/output/campaigns/`
- `backEnd/data/vocabulary.db`
- `backEnd/__pycache__/`、`frontEnd/node_modules/`

历史与收藏**不在** CustomProject 仓库里时，换机器后历史为空，直到在新环境重新生成。

## 九、分发前检查清单

打包给别人前建议：

- [ ] 确认 zip **仅含 CustomProject**，不含 ComfyUI 本体与 `models/` 大文件。
- [ ] 确认对方会把文件夹放到 `ComfyUI/CustomProject/`（不是 ComfyUI 根目录散文件）。
- [ ] **删除或重置** `config/civitai_token.txt`；勿泄露 Token。
- [ ] **重置** `config/auth_store.json`：改管理员密码、清空或换新邀请码。
- [ ] 检查 `config/*.json` 是否含私人提示词、不可外传的内容。
- [ ] `models_manifest.json` 中的 `comfyui_root` 为打包机绝对路径时，**不影响**对方运行（仅记录用），但模型需对方自行下载。
- [ ] 决定是否包含完整 `prompt/jsonData/manifest.json`（体积大但 tag 功能完整）。
- [ ] 方式 B：`npm run build` 后包含 `frontEnd/dist/`。
- [ ] 不包含：`node_modules`、`.venv`、`__pycache__`、`.env`、大量 `frontEnd/dist/assets` 历史重复包（保留一次 build 即可）。

## 十、维护脚本 `scripts/`（可选）

| 脚本 | 作用 |
|------|------|
| `export_models_manifest.py` | 导出本地模型清单到 `config/models_manifest.json` |
| `import_models_from_manifest.py` | 按 manifest 辅助导入 |
| `seed_global_reference_groups.py` | 初始化参考组 |
| `seed_lora_batch_tasks.py` | 示例批量任务 |
| `remove_*_from_manifest.py` | 清理 manifest 中特定 tag/命名 |
| `build_release.ps1` | **闭源一键打包**：前端 build + PyInstaller + zip |

日常使用者可忽略；维护 tag 库或模型清单时使用。
