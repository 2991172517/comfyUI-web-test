# 可组合流水线（待开发）— 方案草案

> 状态：**待开发**  
> 目标：在 Web 端用「预设基础能力模块」拼接成长任务，**不是**复刻 ComfyUI 拖线连节点。

---

## 1. 设计原则

| 原则 | 说明 |
|------|------|
| 模块 = 已审核的工作流变体 | 每个模块对应一个（或一组）固定 API JSON，保证可跑、可维护 |
| 编排 = 前端配置 + 后端状态机 | 用户勾选顺序与参数，后端按步 `queue`，不用任意 DAG |
| 两条技术路子分开 | **扩散**（文生图 / Inpaint / 修脸 / TTP）与 **超分**（RTX）不混在一个 JSON 里 |
| 中间产物可追溯 | 每步输出进历史，下一步可引用上一步文件名 |
| 人工决策优先 | 「要不要重绘 / 再放大」先做用户确认，自动规则后做 |

---

## 2. 基础能力模块（初版清单）

| 模块 ID | 名称 | 工作流变体 | 路子 | 典型输入 | 典型输出 |
|---------|------|------------|------|----------|----------|
| `generate` | 文生图 | `variants/究极参考流` 等 | 扩散 | 提示词、模型、种子 | 成图 |
| `upscale` | 高清放大 | `variants/高清放大_rtx` | 超分 | 上一张图、倍数 | 更大尺寸图 |
| `inpaint` | 局部重绘 | `variants/局部重绘_inpaint` | 扩散 | 原图、蒙版、提示词、denoise | 同尺寸修补图 |
| `face` | 修脸 | （究极流内 FaceDetailer，或未来独立变体） | 扩散 | 图 + 主流程提示词 | 修脸后图 |

后续可扩展：`ttp_refine`（仅分块二采）、`save_only` 等，但 **每增加一项 = 新变体 + 测试 + 文档**。

---

## 3. 预设流水线（Pipeline Preset）示例

用户在前端选 **预设名**，不是自由连线：

### P1 · 究极竖图（现状）

```
generate(究极参考流) → 结束
```

内含：一采 → RTX1.5× → TTP → 修脸 → RTX 1600×2560。

### P2 · 成片微调

```
generate(究极参考流) → user_gate → inpaint(可选) → 结束
```

用户在看预览后决定是否进局部重绘。

### P3 · 先出图再放大

```
generate(简化流/一采) → upscale(×1.5) → user_gate → inpaint(可选)
```

适合不要 TTP/修脸、只要大图时。

### P4 · 仅放大

```
upscale(高清放大_rtx)
```

历史图「拿去超分」。

## 4. 前端交互（拟）

1. **流水线向导页**（或生成页扩展 Tab）  
   - 选预设 P1–P4  
   - 展开每步参数：提示词模块、放大次数（1–2）、是否允许 Inpaint 等  

2. **长任务面板**  
   - 步骤条：生图 ✓ → 超分 ● → 待确认 Inpaint  
   - 每步缩略图 + 「跳过 / 重做 / 进入重绘页」  

3. **模块复用现有页**  
   - Inpaint → `/inpaint`（已存在）  
   - 生图 → `/generate`  
   - 超分 → 待做「高清放大」页或历史一键  

4. **放大次数**  
   - 编排层：`for i in range(n): queue(upscale)`，上一步输出作为下一步 `LoadImage`  
   - 非单一 JSON 内循环  

---

## 5. 后端（拟）

```
POST /api/pipelines/run
{
  "preset_id": "P2",
  "steps_override": [...],
  "generate": { "workflow_id", "overrides", "batch_prompts" },
  "upscale": { "times": 1, "scale": 1.5 },
  "inpaint": { "enabled": false }
}
```

- 表/JSON 存 `pipeline_job_id`、当前步、中间 `image` 文件名  
- 每步完成：WS/轮询 → 写历史 → 触发下一步或 `waiting_user`  
- 与现有 `queue`、`getJob`、额度扣减对齐（按步或按整条扣费待定）

## 6. 动态判断（后期）

| 规则 | 触发 | 动作 |
|------|------|------|
| 长边 &gt; 1536 | 自动 | 建议跳过再次 upscale |
| 人脸检测得分低 | 自动 | 建议走修脸或 Inpaint 脸区 |
| 用户始终可覆盖 | — | 以人工为准 |

---

## 7. 与 ComfyUI 的关系

- ComfyUI：维护 **变体 JSON**、试参、插件。  
- Web：维护 **预设编排 + 用户体验 + 历史串联**。  
- 不在 Web 画节点；**组合表有限**，新增组合 = 产品 + QA 明确加一条 preset。

---

## 8. 实施阶段建议

| 阶段 | 内容 | 优先级 |
|------|------|--------|
| A | 单职责工作流齐备 + 历史「超分 / 重绘」入口 | 高 |
| B | 2～3 个 Pipeline Preset + 后端串行队列 | 中 |
| C | 长任务 UI + 步骤暂停 / 跳过 | 中 |
| D | 自动规则 + 放大次数 UI | 低 |
